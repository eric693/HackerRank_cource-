#11 MEDIUM

Container With Most Water

Array Two Pointers Greedy

題目描述

給定一個長度為 n 的整數陣列 height。有 n 條垂直線，第 i 條線的兩個端點是 (i, 0) 和 (i, height[i])。

找出其中的兩條線，使得它們與 x 軸共同構成的容器可以容納最多的水。

返回容器可以儲存的最大水量。

說明：你不能傾斜容器。

範例

範例 1:

輸入：height = [1,8,6,2,5,4,8,3,7]
輸出：49
解釋：圖中垂直線代表輸入陣列 [1,8,6,2,5,4,8,3,7]。
在此情況下，容器能夠容納水（藍色部分）的最大值為 49。

        |
    |   |   |
    |   |   |   |
    | | | | | | |
  | | | | | | | |
| | | | | | | | |
1 8 6 2 5 4 8 3 7

選擇索引 1 (height=8) 和索引 8 (height=7)：
面積 = min(8, 7) × (8 - 1) = 7 × 7 = 49

範例 2:

輸入：height = [1,1]
輸出：1

限制條件

  • n == height.length
  • 2 <= n <= 105
  • 0 <= height[i] <= 104

解題思路

關鍵觀察

面積 = 寬度 × 高度 = (right - left) × min(height[left], height[right])

雙指標法（推薦）

使用貪心策略：

  1. 從兩端開始，left = 0, right = n - 1
  2. 計算當前面積
  3. 移動較矮的那一邊（因為較矮的邊限制了高度）
  4. 重複直到 left == right

為什麼移動較矮的邊？

  • 如果移動較高的邊，寬度減少且高度不會增加
  • 如果移動較矮的邊，雖然寬度減少但可能找到更高的邊
  • 時間複雜度：O(n)
  • 空間複雜度：O(1)

暴力法（不推薦）

檢查所有可能的配對。

  • 時間複雜度：O(n²)
  • 會超時

程式碼實作

Python 解法

class Solution:
    def maxArea(self, height: List[int]) -> int:
        """
        雙指標法
        
        時間複雜度：O(n)
        空間複雜度：O(1)
        """
        left = 0
        right = len(height) - 1
        max_area = 0
        
        while left < right:
            # 計算當前面積
            width = right - left
            current_height = min(height[left], height[right])
            current_area = width * current_height
            
            # 更新最大面積
            max_area = max(max_area, current_area)
            
            # 移動較矮的那一邊
            if height[left] < height[right]:
                left += 1
            else:
                right -= 1
        
        return max_area


# 方法二：更簡潔的寫法
class Solution2:
    def maxArea(self, height: List[int]) -> int:
        """
        簡潔版雙指標
        """
        left, right = 0, len(height) - 1
        max_area = 0
        
        while left < right:
            # 計算面積並移動指標
            max_area = max(max_area, 
                          min(height[left], height[right]) * (right - left))
            
            if height[left] < height[right]:
                left += 1
            else:
                right -= 1
        
        return max_area

C++ 解法

#include <vector>
#include <algorithm>
using namespace std;

class Solution {
public:
    int maxArea(vector<int>& height) {
        int left = 0;
        int right = height.size() - 1;
        int maxArea = 0;
        
        while (left < right) {
            // 計算當前面積
            int width = right - left;
            int currentHeight = min(height[left], height[right]);
            int currentArea = width * currentHeight;
            
            // 更新最大面積
            maxArea = max(maxArea, currentArea);
            
            // 移動較矮的那一邊
            if (height[left] < height[right]) {
                left++;
            } else {
                right--;
            }
        }
        
        return maxArea;
    }
};

Java 解法

class Solution {
    public int maxArea(int[] height) {
        int left = 0;
        int right = height.length - 1;
        int maxArea = 0;
        
        while (left < right) {
            // 計算當前面積
            int width = right - left;
            int currentHeight = Math.min(height[left], height[right]);
            int currentArea = width * currentHeight;
            
            // 更新最大面積
            maxArea = Math.max(maxArea, currentArea);
            
            // 移動較矮的那一邊
            if (height[left] < height[right]) {
                left++;
            } else {
                right--;
            }
        }
        
        return maxArea;
    }
}

面試要點

常見追問

  • 為什麼移動較矮的邊？
  • 能否用其他方法？
  • 如何證明雙指標的正確性？
  • 與 Trapping Rain Water 有何不同？

解題技巧

  • 理解面積公式
  • 較矮的邊決定高度
  • 貪心策略：移動較矮的邊
  • 一次遍歷即可

常見錯誤

  • 移動指標的邏輯錯誤
  • 高度計算錯誤（用 max 而非 min）
  • 使用 O(n²) 暴力法超時
  • 忘記更新最大面積

算法正確性證明

為什麼移動較矮的邊是最優策略？

反證法：

  1. 假設當前 height[left] < height[right]
  2. 如果我們移動 right（較高的邊）：
    • 寬度減少：right - left 變小
    • 高度不會增加：仍受 height[left] 限制
    • 面積必然不會增加
  3. 如果我們移動 left（較矮的邊）：
    • 寬度減少：right - left 變小
    • 高度可能增加：可能找到更高的邊
    • 面積可能增加

因此，移動較矮的邊是唯一可能找到更大面積的策略。

視覺化示例

height = [1,8,6,2,5,4,8,3,7]

步驟 1: left=0 (h=1), right=8 (h=7)
  面積 = min(1,7) × 8 = 8

步驟 2: left=1 (h=8), right=8 (h=7)  ← 移動left
  面積 = min(8,7) × 7 = 49 ✓ 最大

步驟 3: left=1 (h=8), right=7 (h=3)  ← 移動right
  面積 = min(8,3) × 6 = 18

...繼續直到 left >= right
