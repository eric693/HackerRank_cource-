#1 EASY

Two Sum

Array Hash Table

題目描述

給定一個整數陣列 nums 和一個整數目標值 target，請你在該陣列中找出和為目標值 target 的那兩個整數，並返回它們的陣列索引。

你可以假設每種輸入只會對應一個答案。但是，陣列中同一個元素在答案裡不能重複出現。

你可以按任意順序返回答案。

範例

範例 1:

輸入：nums = [2,7,11,15], target = 9
輸出：[0,1]
解釋：因為 nums[0] + nums[1] == 9，返回 [0, 1]。

範例 2:

輸入：nums = [3,2,4], target = 6
輸出：[1,2]

範例 3:

輸入：nums = [3,3], target = 6
輸出：[0,1]

限制條件

  • 2 <= nums.length <= 104
  • -109 <= nums[i] <= 109
  • -109 <= target <= 109
  • 只會存在一個有效答案

解題思路

方法一：暴力法 (Brute Force)

最直觀的方法是使用兩層迴圈，逐一檢查所有可能的配對。

  • 時間複雜度：O(n²)
  • 空間複雜度：O(1)

方法二：哈希表 (Hash Table) - 推薦

使用哈希表來儲存已經遍歷過的元素及其索引，只需要一次遍歷即可找到答案。

  • 時間複雜度：O(n)
  • 空間複雜度：O(n)

面試重點：面試官通常期望你能想到哈希表的解法，並能夠解釋為什麼這個方法比暴力法更好。

程式碼實作

Python 解法：哈希表（最優解）

class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        """
        使用哈希表來存儲已經遍歷過的數字及其索引
        
        時間複雜度：O(n) - 只需要遍歷一次陣列
        空間複雜度：O(n) - 需要額外的哈希表空間
        """
        # 創建一個字典來存儲數字和它的索引
        num_map = {}
        
        # 遍歷陣列
        for i, num in enumerate(nums):
            # 計算需要找的另一個數字
            complement = target - num
            
            # 如果這個數字已經在字典中，說明找到答案了
            if complement in num_map:
                return [num_map[complement], i]
            
            # 將當前數字和索引存入字典
            num_map[num] = i
        
        # 題目保證有解，所以不會執行到這裡
        return []

C++ 解法：哈希表（最優解）

#include <vector>
#include <unordered_map>
using namespace std;

class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        // 創建一個 unordered_map 來存儲數字和它的索引
        unordered_map<int, int> numMap;
        
        // 遍歷陣列
        for (int i = 0; i < nums.size(); i++) {
            // 計算需要找的另一個數字
            int complement = target - nums[i];
            
            // 如果這個數字已經在 map 中，說明找到答案了
            if (numMap.find(complement) != numMap.end()) {
                return {numMap[complement], i};
            }
            
            // 將當前數字和索引存入 map
            numMap[nums[i]] = i;
        }
        
        // 題目保證有解，所以不會執行到這裡
        return {};
    }
};

Java 解法：哈希表（最優解）

import java.util.HashMap;
import java.util.Map;

class Solution {
    public int[] twoSum(int[] nums, int target) {
        // 創建一個 HashMap 來存儲數字和它的索引
        Map<Integer, Integer> numMap = new HashMap<>();
        
        // 遍歷陣列
        for (int i = 0; i < nums.length; i++) {
            // 計算需要找的另一個數字
            int complement = target - nums[i];
            
            // 如果這個數字已經在 map 中，說明找到答案了
            if (numMap.containsKey(complement)) {
                return new int[] { numMap.get(complement), i };
            }
            
            // 將當前數字和索引存入 map
            numMap.put(nums[i], i);
        }
        
        // 題目保證有解，所以不會執行到這裡
        return new int[] {};
    }
}

面試要點

常見追問

  • 如果陣列是已排序的，有更好的解法嗎？（雙指標法）
  • 如果需要找三個數的和呢？（3Sum 問題）
  • 為什麼使用哈希表比暴力法好？
  • 空間複雜度可以優化到 O(1) 嗎？

面試技巧

  • 先說出暴力法，再優化到哈希表
  • 清楚解釋時間和空間複雜度
  • 考慮邊界條件（如空陣列、只有兩個元素）
  • 可以現場 Live Coding 展示解法

常見錯誤

  • 忘記檢查是否存在於哈希表中就直接訪問
  • 返回了數值而不是索引
  • 使用同一個元素兩次
  • 沒有考慮負數的情況
