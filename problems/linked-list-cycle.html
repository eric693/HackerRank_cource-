#141 EASY

Linked List Cycle

Linked List Two Pointers Hash Table

題目描述

給定一個鏈結串列的頭節點 head，判斷鏈結串列中是否有環。

如果鏈結串列中有某個節點，可以通過連續跟蹤 next 指標再次到達，則鏈結串列中存在環。為了表示給定鏈結串列中的環，我們使用整數 pos 來表示鏈結串列尾部連接到鏈結串列中的位置（索引從 0 開始）。如果 pos 是 -1，則在該鏈結串列中沒有環。

注意：pos 不作為參數傳遞，僅僅是為了標識鏈結串列的實際情況。

如果鏈結串列中存在環，則返回 true。否則，返回 false。

範例

範例 1:

輸入：head = [3,2,0,-4], pos = 1
輸出：true
解釋：鏈結串列中有一個環，其尾部連接到第二個節點。

    3 → 2 → 0 → -4
        ↑         ↓
        ← ← ← ← ←

範例 2:

輸入：head = [1,2], pos = 0
輸出：true
解釋：鏈結串列中有一個環，其尾部連接到第一個節點。

    1 → 2
    ↑   ↓
    ← ←

範例 3:

輸入：head = [1], pos = -1
輸出：false
解釋：鏈結串列中沒有環。

    1

限制條件

  • 鏈結串列中節點的數目範圍是 [0, 104]
  • -105 <= Node.val <= 105
  • pos 為 -1 或者鏈結串列中的一個有效索引

進階要求

你能用 O(1)（即，常數）記憶體解決此問題嗎？

解題思路

方法一：快慢指標（Floyd's Cycle Detection）- 推薦

使用兩個指標，一快一慢：

  • 慢指標：每次移動一步
  • 快指標：每次移動兩步
  • 如果有環，快慢指標最終會相遇
  • 如果無環，快指標會先到達 null
  • 時間複雜度：O(n)
  • 空間複雜度：O(1)

方法二：哈希表

用 Set 記錄訪問過的節點。

  • 時間複雜度：O(n)
  • 空間複雜度：O(n)

程式碼實作

Python 解法

class ListNode:
    def __init__(self, x):
        self.val = x
        self.next = None

class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        """
        快慢指標法（Floyd's Cycle Detection）
        
        時間複雜度：O(n)
        空間複雜度：O(1)
        """
        if not head or not head.next:
            return False
        
        slow = head
        fast = head.next
        
        while slow != fast:
            # 快指標到達末尾，無環
            if not fast or not fast.next:
                return False
            
            slow = slow.next
            fast = fast.next.next
        
        # 快慢指標相遇，有環
        return True


# 方法二：更簡潔的寫法
class Solution2:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        """
        快慢指標 - 簡潔版
        """
        slow = fast = head
        
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            
            if slow == fast:
                return True
        
        return False


# 方法三：哈希表
class Solution3:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        """
        哈希表法
        
        時間複雜度：O(n)
        空間複雜度：O(n)
        """
        visited = set()
        current = head
        
        while current:
            if current in visited:
                return True
            visited.add(current)
            current = current.next
        
        return False

C++ 解法

struct ListNode {
    int val;
    ListNode *next;
    ListNode(int x) : val(x), next(nullptr) {}
};

class Solution {
public:
    bool hasCycle(ListNode *head) {
        if (!head || !head->next) return false;
        
        ListNode *slow = head;
        ListNode *fast = head->next;
        
        while (slow != fast) {
            if (!fast || !fast->next) {
                return false;
            }
            
            slow = slow->next;
            fast = fast->next->next;
        }
        
        return true;
    }
};

Java 解法

class ListNode {
    int val;
    ListNode next;
    ListNode(int x) {
        val = x;
        next = null;
    }
}

class Solution {
    public boolean hasCycle(ListNode head) {
        if (head == null || head.next == null) return false;
        
        ListNode slow = head;
        ListNode fast = head.next;
        
        while (slow != fast) {
            if (fast == null || fast.next == null) {
                return false;
            }
            
            slow = slow.next;
            fast = fast.next.next;
        }
        
        return true;
    }
}

面試要點

常見追問

  • 如何找出環的起始節點？（Linked List Cycle II）
  • 為什麼快慢指標一定會相遇？
  • 能否用 O(1) 空間解決？
  • 如何計算環的長度？

解題技巧

  • 快慢指標是經典技巧
  • 理解「龜兔賽跑」的比喻
  • 注意邊界條件（空串列、單節點）
  • 快指標每次走兩步是關鍵

常見錯誤

  • 忘記檢查 fast.next 是否為空
  • 初始化快慢指標位置錯誤
  • 循環條件寫錯
  • 相遇判斷邏輯錯誤

快慢指標原理

為什麼快慢指標一定會相遇？

假設有環：

  • 慢指標每次移動 1 步
  • 快指標每次移動 2 步
  • 快指標比慢指標多走 1 步
  • 當慢指標進入環後，快指標會逐漸縮小與慢指標的距離
  • 最終快指標會追上慢指標（相遇）

視覺化說明：

環形鏈結串列：
    1 → 2 → 3 → 4 → 5
            ↑       ↓
            8 ← 7 ← 6

初始: slow=1, fast=1
第1步: slow=2, fast=3
第2步: slow=3, fast=5
第3步: slow=4, fast=7
第4步: slow=5, fast=3
第5步: slow=6, fast=5
第6步: slow=7, fast=7 ← 相遇！
