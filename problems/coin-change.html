#322 MEDIUM

Coin Change

Array Dynamic Programming BFS

題目描述

給定不同面額的硬幣 coins 和一個總金額 amount。編寫一個函數來計算可以湊成總金額所需的最少的硬幣個數。如果沒有任何一種硬幣組合能湊成總金額，返回 -1。

你可以認為每種硬幣的數量是無限的。

範例

範例 1:

輸入：coins = [1,2,5], amount = 11
輸出：3
解釋：11 = 5 + 5 + 1

範例 2:

輸入：coins = [2], amount = 3
輸出：-1

範例 3:

輸入：coins = [1], amount = 0
輸出：0

限制條件

  • 1 <= coins.length <= 12
  • 1 <= coins[i] <= 231 - 1
  • 0 <= amount <= 104

解題思路

動態規劃（推薦）

這是一個經典的完全背包問題：

  1. 定義狀態：dp[i] 表示湊成金額 i 所需的最少硬幣數
  2. 狀態轉移：dp[i] = min(dp[i], dp[i - coin] + 1) 對所有硬幣
  3. 初始化：dp[0] = 0，其餘為無限大
  4. 答案：dp[amount]
  • 時間複雜度：O(amount × n) - n 為硬幣種類數
  • 空間複雜度：O(amount)

BFS 解法

將問題視為圖的最短路徑問題，從 0 開始 BFS 到 amount。

  • 時間複雜度：O(amount × n)
  • 空間複雜度：O(amount)

程式碼實作

Python 解法

class Solution:
    def coinChange(self, coins: List[int], amount: int) -> int:
        """
        動態規劃解法
        
        時間複雜度：O(amount × n)
        空間複雜度：O(amount)
        """
        # 初始化 dp 陣列，使用 amount + 1 作為無限大的值
        dp = [amount + 1] * (amount + 1)
        dp[0] = 0
        
        # 對每個金額計算最少硬幣數
        for i in range(1, amount + 1):
            for coin in coins:
                if coin <= i:
                    dp[i] = min(dp[i], dp[i - coin] + 1)
        
        # 如果 dp[amount] 還是無限大，說明無解
        return dp[amount] if dp[amount] != amount + 1 else -1


# 方法二：BFS 解法
from collections import deque

class Solution2:
    def coinChange(self, coins: List[int], amount: int) -> int:
        """
        BFS 解法 - 視為最短路徑問題
        
        時間複雜度：O(amount × n)
        空間複雜度：O(amount)
        """
        if amount == 0:
            return 0
        
        visited = {0}
        queue = deque([(0, 0)])  # (當前金額, 步數)
        
        while queue:
            current_amount, steps = queue.popleft()
            
            for coin in coins:
                next_amount = current_amount + coin
                
                if next_amount == amount:
                    return steps + 1
                
                if next_amount < amount and next_amount not in visited:
                    visited.add(next_amount)
                    queue.append((next_amount, steps + 1))
        
        return -1


# 方法三：記憶化遞迴
class Solution3:
    def coinChange(self, coins: List[int], amount: int) -> int:
        """
        記憶化遞迴
        
        時間複雜度：O(amount × n)
        空間複雜度：O(amount)
        """
        memo = {}
        
        def dp(remaining):
            if remaining == 0:
                return 0
            if remaining < 0:
                return -1
            
            if remaining in memo:
                return memo[remaining]
            
            min_coins = float('inf')
            for coin in coins:
                result = dp(remaining - coin)
                if result >= 0:
                    min_coins = min(min_coins, result + 1)
            
            memo[remaining] = min_coins if min_coins != float('inf') else -1
            return memo[remaining]
        
        return dp(amount)

C++ 解法

#include <vector>
#include <algorithm>
#include <climits>
using namespace std;

class Solution {
public:
    int coinChange(vector<int>& coins, int amount) {
        // 初始化 dp 陣列
        vector<int> dp(amount + 1, amount + 1);
        dp[0] = 0;
        
        // 對每個金額計算最少硬幣數
        for (int i = 1; i <= amount; i++) {
            for (int coin : coins) {
                if (coin <= i) {
                    dp[i] = min(dp[i], dp[i - coin] + 1);
                }
            }
        }
        
        return dp[amount] > amount ? -1 : dp[amount];
    }
};

Java 解法

class Solution {
    public int coinChange(int[] coins, int amount) {
        // 初始化 dp 陣列
        int[] dp = new int[amount + 1];
        Arrays.fill(dp, amount + 1);
        dp[0] = 0;
        
        // 對每個金額計算最少硬幣數
        for (int i = 1; i <= amount; i++) {
            for (int coin : coins) {
                if (coin <= i) {
                    dp[i] = Math.min(dp[i], dp[i - coin] + 1);
                }
            }
        }
        
        return dp[amount] > amount ? -1 : dp[amount];
    }
}

面試要點

常見追問

  • 如何輸出具體使用了哪些硬幣？
  • 如果要求硬幣組合數而非最少數量？
  • 能否用貪心算法？為什麼不行？
  • 空間複雜度能否優化？

解題技巧

  • 識別出這是完全背包問題
  • 理解狀態轉移方程的含義
  • 使用 amount + 1 作為無限大
  • 注意邊界條件（amount = 0）

常見錯誤

  • 初始值設為 -1 而不是無限大
  • 忘記檢查硬幣是否小於當前金額
  • 貪心算法無法得到最優解
  • 沒有處理無解的情況
