#206 EASY

Reverse Linked List

Linked List Recursion

題目描述

給定一個單向鏈結串列的頭節點 head，請你反轉這個鏈結串列，並返回反轉後的鏈結串列。

範例

範例 1:

輸入：head = [1,2,3,4,5]
輸出：[5,4,3,2,1]

範例 2:

輸入：head = [1,2]
輸出：[2,1]

範例 3:

輸入：head = []
輸出：[]

限制條件

  • 鏈結串列的節點數範圍是 [0, 5000]
  • -5000 <= Node.val <= 5000

解題思路

方法一：迭代法（Iterative）- 推薦

使用三個指標來反轉鏈結串列的指向關係。

  • 時間複雜度：O(n)
  • 空間複雜度：O(1)

方法二：遞迴法（Recursive）

使用遞迴的方式來反轉鏈結串列。

  • 時間複雜度：O(n)
  • 空間複雜度：O(n) - 遞迴呼叫堆疊

面試重點：這是經典的鏈結串列操作題，面試官期望你能夠快速寫出迭代解法，並且能夠用遞迴方式實現。

程式碼實作

Python 解法：迭代法

class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

class Solution:
    def reverseList(self, head: ListNode) -> ListNode:
        """
        迭代法反轉鏈結串列
        
        時間複雜度：O(n)
        空間複雜度：O(1)
        """
        # 初始化三個指標
        prev = None
        curr = head
        
        # 遍歷整個鏈結串列
        while curr:
            # 暫存下一個節點
            next_temp = curr.next
            
            # 反轉當前節點的指向
            curr.next = prev
            
            # 移動指標
            prev = curr
            curr = next_temp
        
        # prev 現在指向新的頭節點
        return prev
    
    def reverseListRecursive(self, head: ListNode) -> ListNode:
        """
        遞迴法反轉鏈結串列
        
        時間複雜度：O(n)
        空間複雜度：O(n) - 遞迴呼叫堆疊
        """
        # 基本情況：空串列或只有一個節點
        if not head or not head.next:
            return head
        
        # 遞迴反轉剩餘的串列
        new_head = self.reverseListRecursive(head.next)
        
        # 反轉當前節點和下一個節點的指向
        head.next.next = head
        head.next = None
        
        return new_head

C++ 解法：迭代法

struct ListNode {
    int val;
    ListNode *next;
    ListNode() : val(0), next(nullptr) {}
    ListNode(int x) : val(x), next(nullptr) {}
    ListNode(int x, ListNode *next) : val(x), next(next) {}
};

class Solution {
public:
    ListNode* reverseList(ListNode* head) {
        // 初始化三個指標
        ListNode* prev = nullptr;
        ListNode* curr = head;
        
        // 遍歷整個鏈結串列
        while (curr != nullptr) {
            // 暫存下一個節點
            ListNode* nextTemp = curr->next;
            
            // 反轉當前節點的指向
            curr->next = prev;
            
            // 移動指標
            prev = curr;
            curr = nextTemp;
        }
        
        // prev 現在指向新的頭節點
        return prev;
    }
    
    ListNode* reverseListRecursive(ListNode* head) {
        // 基本情況：空串列或只有一個節點
        if (head == nullptr || head->next == nullptr) {
            return head;
        }
        
        // 遞迴反轉剩餘的串列
        ListNode* newHead = reverseListRecursive(head->next);
        
        // 反轉當前節點和下一個節點的指向
        head->next->next = head;
        head->next = nullptr;
        
        return newHead;
    }
};

Java 解法：迭代法

class ListNode {
    int val;
    ListNode next;
    ListNode() {}
    ListNode(int val) { this.val = val; }
    ListNode(int val, ListNode next) { this.val = val; this.next = next; }
}

class Solution {
    public ListNode reverseList(ListNode head) {
        // 初始化三個指標
        ListNode prev = null;
        ListNode curr = head;
        
        // 遍歷整個鏈結串列
        while (curr != null) {
            // 暫存下一個節點
            ListNode nextTemp = curr.next;
            
            // 反轉當前節點的指向
            curr.next = prev;
            
            // 移動指標
            prev = curr;
            curr = nextTemp;
        }
        
        // prev 現在指向新的頭節點
        return prev;
    }
    
    public ListNode reverseListRecursive(ListNode head) {
        // 基本情況：空串列或只有一個節點
        if (head == null || head.next == null) {
            return head;
        }
        
        // 遞迴反轉剩餘的串列
        ListNode newHead = reverseListRecursive(head.next);
        
        // 反轉當前節點和下一個節點的指向
        head.next.next = head;
        head.next = null;
        
        return newHead;
    }
}

面試要點

常見追問

  • 能否用遞迴實現？
  • 如何反轉部分鏈結串列（從位置 m 到 n）？
  • 如何判斷反轉是否成功？
  • 空間複雜度能否優化到 O(1)？

面試技巧

  • 畫圖解釋指標的移動過程
  • 注意邊界條件（空串列、單節點）
  • 迭代法是最常見的解法
  • 能同時寫出迭代和遞迴兩種解法更佳

常見錯誤

  • 忘記暫存 next 指標導致鏈結串列斷裂
  • 返回錯誤的頭節點
  • 沒有處理空串列的情況
  • 遞迴時忘記設置 head.next = None
